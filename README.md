# Sustainability Software Engineering Education Artifacts

This repository contains anonymized instructional artifacts for a project-based Software Engineering education experience focused on sustainability-oriented requirements.

The materials are intended to support transparency, review, and possible adaptation by other educators. They include activity descriptions, classroom templates, review instruments, UML modeling instructions, presentation and reflection prompts, and a sustainability-oriented requirements template.

Student-produced artifacts are not included in their original form because they may contain contextual information related to students, stakeholders, projects, or institutional settings. 

## Repository structure

| File | Description |
|---|---|
| `01_course_context.md` | Summarizes the anonymized educational context of the experience. |
| `02_product_vision_activity.md` | Describes the product vision activity and its expected elements. |
| `03_product_backlog_activity.md` | Describes the product backlog activity, user story format, and acceptance criteria. |
| `04_uml_modeling_activity.md` | Describes the UML modeling activity and expected diagrams. |
| `05_sustainability_requirements_activity.md` | Presents the sustainability-oriented requirements activity. |
| `06_product_vision_self_assessment.md` | Provides an anonymized version of the product vision self-assessment instrument. |
| `07_product_backlog_self_assessment.md` | Provides an anonymized version of the product backlog self-assessment instrument. |
| `08_uml_and_sustainability_review_instruments.md` | Provides complementary review instruments for UML models and sustainability-oriented requirements. |
| `09_presentation_and_reflection_prompts.md` | Provides prompts for presentation, classroom discussion, and reflection. |
| `10_anonymized_or_synthetic_examples.md` | Provides anonymized, generalized, or synthetic examples. |
| `11_anonymization_notes.md` | Explains the anonymization decisions adopted in the repository. |
