# Product Vision Activity

## Objective

The objective of this activity is to support students in framing the problem, identifying target users and stakeholders, defining the expected value of the solution, and clarifying assumptions, constraints, risks, and scope.

## Learning goals

After completing this activity, students are expected to be able to:

- describe the problem addressed by a software product;
- identify users and stakeholders;
- explain the value proposition of the proposed solution;
- define the initial scope of the product;
- identify assumptions, constraints, risks, and out-of-scope elements;
- connect the product vision to later requirements and design activities.

## Student task

Each team should prepare a product vision document addressing the following elements:

1. Problem addressed by the project
2. Target users
3. Main stakeholders
4. Expected value of the solution
5. Main features
6. Assumptions
7. Constraints
8. Risks
9. Out-of-scope elements

## Suggested template

| Element | Description |
|---|---|
| Problem | What problem does the product address? |
| Target users | Who will use the product directly? |
| Stakeholders | Who is affected by the product or has an interest in it? |
| Value proposition | What value does the product intend to deliver? |
| Main features | What are the main expected capabilities of the product? |
| Assumptions | What assumptions are being made about the context, users, or technology? |
| Constraints | What limitations or restrictions should be considered? |
| Risks | What risks may affect the product, users, or development process? |
| Out of scope | What will not be addressed in this version of the product? |

## Review questions

Students should review their product vision using the following questions:

- Is the problem clearly described?
- Are the target users and stakeholders identified?
- Is the expected value of the solution explicit?
- Are the main features coherent with the problem?
- Are assumptions, constraints, and risks described?
- Are out-of-scope elements clearly defined?
- Does the product vision provide enough information to guide the product backlog?
